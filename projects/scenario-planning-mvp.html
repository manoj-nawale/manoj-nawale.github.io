Scenario Planning MVP (Airflow + MinIO)

A data-engineering-first decision support pipeline for reproducible what-if scenario analysis. The workflow generates a deterministic baseline, applies scenario knobs (demand/variable/logistics), runs a capacity-aware toy solver, and publishes auditable outputs to MinIO by dt/run_id/scenario_id.

What it solves

Planning teams need quick scenario comparisons without ad-hoc spreadsheet drift. This MVP turns scenario simulation into a repeatable, traceable run pipeline with consistent run metadata.

How to run

docker compose up -d
# open http://localhost:8080 (admin/admin)
# trigger:
#   hello_scenario_planning (optional)
#   minio_smoketest (optional)
#   scenario_planning_mvp (main)
# verify outputs in MinIO bucket 'scenarioops' at http://localhost:9001

Repository

github.com/manoj-nawale/scenario-planning-mvp

Key implementation highlights

  • Run-partitioned MinIO layout for reproducibility and auditability.
  • Three implemented knobs: demand scaling, variable cost scaling, logistics cost scaling.
  • Published artifacts: fact_production.json, kpi_total.json, kpi_cost_breakdown.json, and run metadata.
  • Local stack uses Airflow + MinIO + Postgres via Docker Compose.
